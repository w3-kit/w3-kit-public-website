# Configuration

## `w3-kit.config.ts`

The root configuration file lets you customize chains, features, and theming at the project level. Create it in your project root:

```ts
import { defineConfig } from "w3-kit/config";

export default defineConfig({
  chains: ["ethereum", "polygon", "arbitrum", "solana"],
  features: {
    swap: true,
    bridge: true,
    nft: true,
    staking: false,
  },
  theme: {
    accent: "#6366F1",
    radius: "0.75rem",
    fontFamily: "Inter, system-ui, sans-serif",
  },
  rpc: {
    ethereum: process.env.VITE_ETH_RPC_URL,
    polygon: process.env.VITE_POLYGON_RPC_URL,
  },
});
```

## Environment Variables

Store RPC URLs and API keys in a `.env` file. Vite exposes variables prefixed with `VITE_` to the browser:

```env
VITE_ETH_RPC_URL=https://eth-mainnet.g.alchemy.com/v2/your-key
VITE_POLYGON_RPC_URL=https://polygon-mainnet.g.alchemy.com/v2/your-key
VITE_SOLANA_RPC_URL=https://api.mainnet-beta.solana.com
VITE_WALLETCONNECT_PROJECT_ID=your-project-id
```

Never commit `.env` files. Add `.env` to your `.gitignore` and use `.env.example` as a template.

## Provider Setup

Wrap your application with the necessary providers in `src/app/main.tsx`:

```tsx
import React from "react";
import ReactDOM from "react-dom/client";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { WagmiProvider, createConfig, http } from "wagmi";
import { mainnet, polygon } from "wagmi/chains";
import App from "./App";
import "w3-kit/styles";

const wagmiConfig = createConfig({
  chains: [mainnet, polygon],
  transports: {
    [mainnet.id]: http(import.meta.env.VITE_ETH_RPC_URL),
    [polygon.id]: http(import.meta.env.VITE_POLYGON_RPC_URL),
  },
});

const queryClient = new QueryClient();

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <WagmiProvider config={wagmiConfig}>
      <QueryClientProvider client={queryClient}>
        <App />
      </QueryClientProvider>
    </WagmiProvider>
  </React.StrictMode>,
);
```

## Solana Provider Setup

For Solana-enabled projects, add the wallet adapter providers:

```tsx
import {
  ConnectionProvider,
  WalletProvider,
} from "@solana/wallet-adapter-react";
import { PhantomWalletAdapter } from "@solana/wallet-adapter-wallets";
import { clusterApiUrl } from "@solana/web3.js";

const endpoint = import.meta.env.VITE_SOLANA_RPC_URL ?? clusterApiUrl("mainnet-beta");
const wallets = [new PhantomWalletAdapter()];

function SolanaProviders({ children }: { children: React.ReactNode }) {
  return (
    <ConnectionProvider endpoint={endpoint}>
      <WalletProvider wallets={wallets} autoConnect>
        {children}
      </WalletProvider>
    </ConnectionProvider>
  );
}
```

## TypeScript Path Aliases

The default `tsconfig.json` includes the `~/` path alias for `src/`:

```json
{
  "compilerOptions": {
    "paths": {
      "~/*": ["./src/*"]
    }
  }
}
```

This lets you import using `~/entities/wallet` instead of relative paths like `../../../entities/wallet`.

## Vite Configuration

The `vite.config.ts` generated by `w3-kit init` includes `tsconfigPaths: true` to enable the path aliases at build time. If you are adding w3-kit to an existing project, ensure the same is configured:

```ts
import { defineConfig } from "vite";
import viteReact from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  resolve: {
    tsconfigPaths: true,
  },
  plugins: [viteReact(), tailwindcss()],
});
```
